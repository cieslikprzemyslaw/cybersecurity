## Zakres

Celem nie było zostanie exploit developerem ani nauka zaawansowanych łańcuchów gadgetów.

Praktyka skupiała się na:

- rozpoznawaniu serializowanych danych,
- wskazywaniu granic zaufania,
- odróżnieniu kodowania od ochrony integralności,
- zmianie jednej właściwości istotnej z punktu widzenia bezpieczeństwa,
- sprawdzeniu, czy zmieniony stan został zaakceptowany,
- potwierdzeniu wpływu przez zachowanie chronionego endpointu,
- zaproponowaniu bezpieczniejszego projektu po stronie serwera.
