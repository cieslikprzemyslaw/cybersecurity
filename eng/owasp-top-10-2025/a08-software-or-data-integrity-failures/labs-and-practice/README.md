## Scope

The purpose was not to become an exploit developer or study advanced gadget chains.

The completed practice focused on:

- recognising serialized data,
- identifying trust boundaries,
- distinguishing encoding from integrity protection,
- modifying one security-sensitive property,
- observing whether altered state was accepted,
- proving impact through protected server behaviour,
- recommending safer server-side design.
