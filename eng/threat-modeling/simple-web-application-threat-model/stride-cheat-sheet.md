# STRIDE Cheat Sheet

- **Spoofing** - pretending to be another user, service, or system
- **Tampering** - modifying data, requests, messages, or stored information
- **Repudiation** - denying an action when the system lacks reliable audit evidence
- **Information Disclosure** - exposing data to an unauthorised party
- **Denial of Service** - making a system or function unavailable
- **Elevation of Privilege** - gaining permissions beyond those originally granted

Use STRIDE as a checklist for each actor, process, data store, data flow, and trust boundary. A STRIDE entry is a potential threat, not proof of a vulnerability.
